Authentication & Security

JWT Bearer Token Authentication

Most API endpoints require authentication via JWT tokens. Include the token in the Authorization header:

Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...

CSRF Protection

All POST, PUT, and DELETE requests require CSRF tokens. Get a token from:

GET /csrf_token.php

Include the token in the header:

X-CSRF-Token: your-csrf-token-here

Rate Limiting

API requests are rate limited to 100 requests per minute per IP address.

Security Headers

All responses include comprehensive security headers for protection against common attacks.

Integration Examples

JavaScript/Fetch API

// Get CSRF token const csrfResponse = await fetch('/csrf_token.php'); const csrfData = await csrfResponse.json(); // Login request const loginResponse = await fetch('/api/auth/login', { method: 'POST', headers: { 'Content-Type': 'application/json', 'X-CSRF-Token': csrfData.token }, body: JSON.stringify({ username: 'your-username', password: 'your-password' }) }); const loginData = await loginResponse.json(); console.log('Login successful:', loginData);

cURL Examples

# Get CSRF token CSRF_TOKEN=$(curl -s http://localhost/csrf_token.php | jq -r '.token') # Login with CSRF protection curl -X POST http://localhost/api/auth/login \ -H "Content-Type: application/json" \ -H "X-CSRF-Token: $CSRF_TOKEN" \ -d '{"username": "admin", "password": "Passw0rd!"}'

PHP Integration

// Using the Auth class directly require_once 'lib/Auth.php'; $auth = new Auth(); $result = $auth->login([ 'username' => 'admin', 'password' => 'Passw0rd!' ]); if ($result['success']) { $token = $result['tokens']['accessToken']; // Use token for subsequent requests }

Response Formats

All API responses follow a consistent format:

// Success Response { "success": true, "data": { ... }, "metadata": { ... } } // Error Response { "success": false, "error": "Error message description", "code": "ERROR_CODE" }

Status Codes

  • 200 OK - Request successful
  • 400 Bad Request - Invalid request parameters
  • 401 Unauthorized - Authentication required
  • 403 Forbidden - Insufficient permissions
  • 404 Not Found - Resource not found
  • 429 Too Many Requests - Rate limit exceeded
  • 500 Internal Server Error - Server error
